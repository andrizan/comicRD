# ComicRD Flutter

ComicRD Flutter is the rewrite of ComicRD from a Tauri + React desktop app into a Flutter desktop app backed by reusable Rust core logic.

This repository is intentionally structured around a new runtime:

```text
Tauri + React + WebView
↓
Flutter desktop + Dart UI + Rust core via flutter_rust_bridge
```

The old Tauri shell, React components, WebView image protocol, and Tauri commands are not part of this app. Rust remains responsible for the heavy application logic: library scanning, ZIP/CBZ handling, SQLite metadata, reader state, image resizing, caching, and backup/import.

## Status

This rewrite is in progress.

Implemented so far:

- Flutter desktop project under `app_flutter/`
- Rust workspace under `crates/`
- `comicrd_core` with database migrations, library/chapter discovery, reader state, image rendering, cache, scan state, bookmarks, favorites, history, and backup/import tests
- `comicrd_bridge` with `flutter_rust_bridge` generated bindings
- Dart bridge entrypoint and `ComicRdApi` facade
- Flutter shell with routing, theme, locale state, library page, comic detail page, settings panel, and reader image rendering

Still in progress:

- Packaging and release workflow
- Manual desktop smoke testing on Windows and macOS

## Prerequisites

- Flutter desktop SDK
- Dart SDK from Flutter
- Rust toolchain
- `flutter_rust_bridge_codegen` matching the runtime dependency, currently `2.12.0`
- `cargo-expand`, required by FRB codegen
- Platform desktop build tools for Linux, Windows, or macOS

Install FRB codegen:

```bash
cargo install flutter_rust_bridge_codegen --version 2.12.0
```

## Development Commands

From the repository root:

```bash
cargo test
flutter_rust_bridge_codegen generate --config-file flutter_rust_bridge.yaml
```

From `app_flutter/`:

```bash
flutter pub get
flutter analyze
flutter test
flutter run -d linux
```

Desktop build commands:

```bash
# Linux
flutter build linux --release

# Windows, from a Windows host with Visual Studio desktop build tools
flutter build windows --release

# macOS, from a macOS host with Xcode
flutter build macos --release
```

Desktop app metadata is configured as:

```text
App name: ComicRD
Bundle/application ID: com.andrizan.comicrd
```

In this Codex workspace, shell commands are expected to be prefixed with `rtk`. For example:

```bash
rtk cargo test
rtk flutter analyze
```

## Bridge Workflow

The bridge API boundary lives in:

```text
crates/comicrd_bridge/src/api.rs
```

Generated files are:

```text
crates/comicrd_bridge/src/frb_generated.rs
app_flutter/lib/api.dart
app_flutter/lib/frb_generated.dart
app_flutter/lib/frb_generated.io.dart
```

`app_flutter/lib/bridge_generated.dart` is a stable export file for Flutter code. UI code should prefer `app_flutter/lib/api/comicrd_api.dart` instead of calling generated FRB functions directly.

Regenerate the bridge after changing public bridge structs or functions:

```bash
flutter_rust_bridge_codegen generate --config-file flutter_rust_bridge.yaml
```

## Architecture Notes

- Flutter owns UI state, routes, theme, localization, widgets, and desktop presentation.
- Rust core owns SQLite, filesystem discovery, image IO, resizing, page caches, scan state, progress, bookmarks, history, and backup/import.
- `comicrd_core` must not depend on Tauri, Flutter, or generated FRB code.
- `comicrd_bridge` is the only Rust crate that knows about `flutter_rust_bridge`.
- Generated FRB files should be committed because the Flutter analyzer and app source depend on them.

## Verification

Before committing implementation work, run the relevant checks:

```bash
cargo test
flutter analyze
flutter test
```

If only Rust core behavior changed, `cargo test` is the minimum check. If Dart, Flutter UI, generated bridge files, or `pubspec.yaml` changed, run Flutter analyzer and tests as well.
